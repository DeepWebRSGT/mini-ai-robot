1.1 — Raspberry Pi Imager'ı İndir ve Çalıştır

Adres: raspberrypi.com/software

İndir → Kur → Çalıştır

Raspberry Pi Device → Raspberry Pi 4
Operating System → Raspberry Pi OS (64-bit) → Raspberry Pi OS Lite
Not: Lite = masaüstü yok, sadece terminal. Daha hızlı ve az RAM tüketir.
Eğer HDMI monitör kullanacaksan → Raspberry Pi OS with Desktop

1.3 — Ön Yapılandırma (En Önemli Adım!)

"Edit Settings" butonuna tıkla. Şu ayarları YAP:

General tab:
· Hostname: mini-ai-robot
· Username: pi
· Password: kendin_belirle
· WiFi SSID: (evinin adı)
· WiFi Password: (şifren)

Services tab:
· SSH: ☑ Enable SSH → Use password authentication

NOT: SSH'yi aktif et ki kafan sıkışınca SSH ile bağlanabilesin.

1.4 — SD Karta Yaz

SD kartı bilgisayara tak → "YES" yazılım formatlayacak → bekle (~5-10 dk)
SD kart çıktı → Pi'ye tak → güç ver → yeşil LED yanıp sönmeye başlar
🔍 KONTROL: SSH ile bağlan: ssh pi@mini-ai-robot.local veya ssh pi@192.168.x.x (IP adresini router'dan bul)

⏱️ 15 dk

Pi'yi ilk kez aç, temel ayarları yap, donanım arayüzlerini etkinleştir.

2.1 SSH ile Bağlan (veya monitör+klavye kullan)

$ ssh pi@mini-ai-robot.local
# Şifreni gir → Pi'nin terminalini görüyorsun

2.2 Sistemi Güncelle

$ sudo apt update && sudo apt upgrade -y
$ sudo apt install -y git curl wget unzip i2c-tools
Bu ~5-10 dk sürebilir. İnternet hızına bağlı.

2.3 SPI, I2C ve I2S'yi Etkinleştir (raspi-config)

$ sudo raspi-config

Menüde sırayla şunları yap:

1. Interface Options → SPI → Enable
2. Interface Options → I2C → Enable
3. Interface Options → SSH → Enable (zaten açık olmalı)
4. Interface Options → Camera → Enable (opsiyonel)
5. Performans Options → GPU Memory → 128 (AI için daha çok CPU RAM'i)
6. Finish → Reboot

2.4 I2S Ses Arayüzünü Aktif Et (En Kritik Adım)

$ sudo nano /boot/firmware/config.txt

Dosyanın en altına şunları EKLE:

# === Mini AI Robot — Donanım Ayarları ===
dtoverlay=googlevoicehat-codec
dtparam=i2s=on
dtparam=spi=on
dtparam=spi_speed=32000000
dtparam=i2c_arm=on
gpu_mem=128
gpio=17=ip,pd
gpio=22=op,dh
# Kamera varsa:
# dtoverlay=imx708

Kaydet: Ctrl+X → Y → Enter

$ sudo reboot
Reboot sonrası SSH tekrar bağlan: ls /dev/i2c-* → /dev/i2c-1 görmelisin
arecord -l → **** List of CAPTURE Hardware Devices **** (ses kartı listelenmeli)
⚠️ SIK KARŞILAŞILAN HATA: Eğer /boot/firmware/config.txt yoksa /boot/config.txt'yi dene (eski Pi OS sürümleri).
ls /boot/ ile hangi klasör olduğunu kontrol et.

🔌 Breadboard Test — Ses Sistemi (I2S)
⏱️ 30 dk

Henüz lehimleme YOK. Her şeyi breadboard'da kur, test et, sonra kalıcı yap.

3.1 INMP441 Mikrofon Bağlantısı

INMP441'in 6 pini var. Şöyle bağla:

INMP441 Pini→Pi GPIOKablo Rengi
VDD (3.3V)→Pin 1 (3.3V)🔴 Kırmızı
GND→Pin 6 (GND)⚫ Siyah
SCK (I2S Clock)→Pin 12 (GPIO18)🟡 Sarı
WS (Word Select)→Pin 35 (GPIO19)🟡 Sarı
SD (Serial Data)→Pin 38 (GPIO20)🟡 Sarı
L/R (Left/Right)→GND (sol kanal)⚫ Siyah

3.2 MAX98357A Amplifikatör Bağlantısı

MAX98357A Pini→Pi GPIOKablo Rengi
VIN (5V)→Pin 2 (5V)🔴 Kırmızı
GND→Pin 14 (GND)⚫ Siyah
BCLK→Pin 12 (GPIO18)🟡 Sarı
LRC→Pin 35 (GPIO19)🟡 Sarı
DIN→Pin 40 (GPIO21)🟡 Sarı
ÖNEMLİ: INMP441 ve MAX98357A aynı BCLK (GPIO18) ve LRCLK (GPIO19) pinlerini PAYLAŞIR. Bu NORMALDIR. I2S veri yolu böyle çalışır.

3.3 Speaker'ı Amplifikatöre Bağla

MAX98357A→Speaker
OUT+→Speaker (+) kırmızı kablo
OUT-→Speaker (-) siyah kablo
İPUCU: Speaker'ın hangi ucu + hangisi - önemli değil. Her iki türlü de çalar (ters bağlarsan ses faz dışı kalır ama duyulur).

3.4 Ses Testi — Mikrofon Kaydı

$ sudo apt install -y alsa-utils
$ arecord -l
# kart numarasını not et (genelde 0 veya 1)
$ arecord -d 5 -f S16_LE -r 16000 -c 1 test.wav
# 5 saniye kaydeder. Mikrofona konuş!
test.wav dosyası oluştu. Oynat: aplay test.wav — kendi sesini duymalısın.

3.5 Ses Testi — Hoparlör Çıkışı

$ speaker-test -t sine -f 440 -l 1
# 440Hz (La notası) test sesi duymalısın
Hoparlörden 1 saniyelik "beeep" sesi gelir. Ses yoksa bağlantıları kontrol et.

ILI9341 Pini→Pi GPIOKablo

🔘 Buton + LED Testi
⏱️ 10 dk

Bas-konuş butonu ve durum LED'ini bağla, çalıştığını doğrula.

8.1 Buton Bağlantısı

Buton→Pi GPIO
Bir Uç→Pin 11 (GPIO17)
Diğer Uç→Pin 14 (GND)
İç Pull-Up: GPIO17'nin içinde 10kΩ direnç var. Harici direnç GEREKMEZ. Butona basınca GPIO GND'ye çekilir.

8.2 LED Bağlantısı

LED→Pi GPIO
Anot (+ uzun bacak)→220Ω direnç → Pin 15 (GPIO22)
Katot (- kısa bacak)→Pin 34 (GND)
⚠️ 220Ω DİRENÇ KULLANMAMAK GPIO'YU YAKAR! LED'ler 20mA çeker, GPIO maks 16mA verir. Direnç şart.

🏗️ Son Montaj — Her Şeyi Birleştir
⏱️ 45 dk

Artık her parça breadboard'da çalıştı. Şimdi kalıcı montaja geç.

9.1 Breadboard'dan Çıkarma Sırası

  1. Gücü KES (Pi'yi kapat, güç kablosunu çek)
  2. Her bağlantının FOTOĞRAFINI ÇEK (sonra başvurmak için)
  3. Kabloları breadboard'dan TEK TEK sök
  4. Her kabloyu olduğu gibi bırak (uçları aynı)

9.2 Montaj Sırası (Kutuya)

  1. Pi 4'ü kutu tabanına vidalarla sabitle (M2.5 vida + ara parça)
  2. PowerBoost 1000C → Pi'nin yanına (M2.5 vidalar veya çift taraflı bant)
  3. LiPo batarya → en alta, hareket etmeyecek şekilde
  4. SPDT şalter → kutunun yan yüzüne (PowerBoost ile batarya arası)
  5. ILI9341 ekran → kutunun ön yüzüne (ya 3D baskıya gömülü, ya çift taraflı bant)
  6. INMP441 mikrofon → kutunun üst/ön kısmında, ses alacak şekilde (küçük delik aç)
  7. MAX98357A + speaker → yan veya alt kısımda
  8. Buton → yan yüzde, kolay erişilebilir yerde
  9. LED → ön yüzde (görülebilecek yerde)
  10. Kablo yönetimi → kabloları bük, zip tie ile topla

9.3 Bağlantıları Sıkılaştır

Dupont kablolar zamanla gevşeyebilir. Şunları yap:

  • Kritik bağlantıları (güç, I2S) sıcak silikonla sabitle
  • Kabloları kısalt (gereksiz uzunlukları kes)
  • Her bağlantıyı çektirerek test et (gevşek var mı?)
  • Kısa devre riskine karşı açıkta kalan pinleri kontrol et
⚠️ İLK AÇILIŞ: Kutuya monte ettikten sonra ÖNCE güç şalterini aç, PowerBoost'un LED'i yanmalı (şarj/çalışma). Sonra Pi açılır. İlk seferde ekranı, sesi, butonu tekrar test et.

🔋 Güç Yönetimi (Batarya)
⏱️ 15 dk

10.1 Kablo Bağlantı Şeması (Güç)

LiPo (+) ── SPDT Şalter ── PowerBoost Bat(+) ── PowerBoost 5V ── Pi USB-C (5V pin)
LiPo (-) ───────────────── PowerBoost Bat(-) ── PowerBoost GND ── Pi GND

10.2 Şarj Etme

  • PowerBoost 1000C'nin USB-C portuna mikroUSB kablo tak
  • Şarj olurken PowerBoost'taki LED yanar
  • Tam dolunca LED söner (3000mAh için ~3 saat)
  • Şarj ederken robot KAPALI olmalı (şalter OFF)

10.3 Pil Ömrü

3000mAh · Pi 4 idle: ~600mA · Pi 4 AI çalışırken: ~900-1200mA

Tahmini:
· Bekleme (buton bekleme): ~5 saat
· Aktif kullanım (konuşma): ~2-3 saat
· LLM çalışırken (4GB Pi): ~1.5-2 saat
⚠️ LiPo GÜVENLİĞİ:
· LiPo bataryayı DELME, EZME, ISLATMA
· Şişerse hemen kullanmayı bırak, güvenli şekilde at
· LiPo şarj için ÖZEL şarj cihazı kullan (PowerBoost'un içinde koruma var)
· Kısa devre yapma — yangın çıkabilir!

🔧 Sorun Giderme
⏱️ Her zaman

SORUN Hoparlörden ses gelmiyor

1. speaker-test -t sine -f 440 -l 1 dene → ses yoksa bağlantıyı kontrol et
2. MAX98357A 5V ile besleniyor mu? (Pin 2)
3. BCLK/LRC/DIN doğru pinlerde mi?
4. dtoverlay=googlevoicehat-codec config.txt'de var mı?
5. Ses seviyesi: alsamixer → F6 → seç "googlevoicehat" → yukarı ok tuşuyla sesi aç

SORUN Vosk "Model bulunamadı" hatası

1. Model indi mi? ls /home/pi/mini-ai-robot/models/
2. config.py'de model_path doğru yolu gösteriyor mu?
3. Elle dene: python -c "from vosk import Model; m=Model('models/vosk-model-small-nl-0.22'); print('OK')"

SORUN Ekran beyaz / boş

1. ls /dev/spi* → spi cihazları görünüyor mu?
2. raspi-config → SPI etkin mi?
3. CS pinini kontrol et (CE0 = Pin 24, GPIO8)
4. Ekrana 3.3V gidiyor mu? (5V takarsan ekranı yakabilirsin)

SORUN Buton çalışmıyor

1. GPIO17'ye pull-up var mı? (GPIO.setup(17, GPIO.IN, pull_up_down=GPIO.PUD_UP))
2. Butonun diğer ucu GND'de mi?
3. python -c "import RPi.GPIO as g; g.setmode(g.BCM); g.setup(17,g.IN,pull_up_down=g.PUD_UP); print(g.input(17))"
→ basılı değilken 1, basılıyken 0 görmelisin

SORUN Ollama çalışmıyor / çok yavaş

1. sudo systemctl status ollama → çalışıyor mu?
2. 4GB Pi kullanıyor musun? 2GB Pi'de LLM çalışmaz
3. İlk soru her zaman yavaştır (model RAM'e yüklenir)
4. Daha küçük model dene: ollama pull tinyllama:1.1b-q4 (daha hızlı ama daha az zeki)

SORUN "ImportError: No module named 'board'"

pip install adafruit-blinka yap. Hala olmazsa: pip install adafruit-circuitpython-rgb-display

SORUN RPi.GPIO hatası

Kitaptaki kodlar bilgisayarında test edildiği için RPi.GPIO bulamaz. Bu NORMALDİR. Sadece Pi'de çalışır. Bilgisayarda test için GPIO hatalarını yoksayabilirsin.

SORUN "arecord: device not found"

1. dtoverlay=googlevoicehat-codec config.txt'ye ekledin mi? reboot yaptın mı?
2. lsmod | grep snd → ses modülleri yüklü mü?
3. sudo nano /etc/modules → alt satıra snd-soc-googlevoicehat-codec ekle

⚡ Otomatik Başlatma (Her Açılışta Robot Çalışsın)
⏱️ 10 dk

Pi'yi açtığında robotun kendiliğinden başlaması için systemd servisi kur.

13.1 Servis Dosyası Oluştur

$ sudo nano /etc/systemd/system/mini-ai-robot.service

İçine şunu yapıştır:

[Unit]
Description=Mini AI Robot
After=network.target sound.target

[Service]
Type=simple
User=pi
WorkingDirectory=/home/pi/mini-ai-robot
Environment="PATH=/home/pi/mini-ai-robot/robot-env/bin:/usr/bin:/bin"
ExecStart=/home/pi/mini-ai-robot/robot-env/bin/python src/main.py
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target

Kaydet: Ctrl+X → Y → Enter

13.2 Servisi Aktif Et

$ sudo systemctl daemon-reload
$ sudo systemctl enable mini-ai-robot.service
$ sudo systemctl start mini-ai-robot.service

13.3 Kontrol

$ sudo systemctl status mini-ai-robot.service
"active (running)" yazmalı. Hata varsa: sudo journalctl -u mini-ai-robot.service -n 20 ile logları gör.

13.4 Durdurma / Yeniden Başlatma

$ sudo systemctl stop mini-ai-robot.service # Durdur
$ sudo systemctl restart mini-ai-robot.service # Yeniden başlat
$ sudo systemctl disable mini-ai-robot.service # Otomatik başlamasını kapat
